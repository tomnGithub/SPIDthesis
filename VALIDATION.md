## Missing keyword EditorID creation

This revision adds creation of output `KYWD` records for distributed Keyword rules that supply an unresolved bare EditorID. The implementation was checked for these cases:

- An EditorID that already resolves to a winning Keyword reuses the existing FormKey.
- An unresolved bare EditorID calls `PatchMod.Keywords.AddNew(editorId)` once, then immediately registers the result in the in-memory record index.
- Later rules using the same EditorID resolve to the newly created FormKey, including case-insensitive matches.
- Keyword dependency ordering and later NPC string filters can retrieve the generated EditorID from the updated index.
- Explicit `0xFORMID‾Plugin`, `FORMID:Plugin`, plugin-looking values, whitespace-containing values, and malformed `0x...` tokens are not converted into new keywords.
- Disabled Keyword processing does not create output keyword records.

Mutagen's current group API documents `mod.<Group>.AddNew(editorId)` as the supported route for creating a new record with an allocated FormKey and supplied EditorID.
